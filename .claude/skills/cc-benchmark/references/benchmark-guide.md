## 7 步执行模板

### ① 定标杆（Anchor）模板

**标杆三要素检查表**：
```
[ ] 具体：标杆是哪个具体对象？（不是"行业领先"，是"X 文档/Y 工具/Z 项目"）
[ ] 可量化：能量化对比吗？（行数/章节/特征/性能指标）
[ ] 可获取：能拿到原文做逐项对照吗？（PDF/仓库/可访问 URL）
```

**标杆画像**（填空）：
```
标杆对象：{具体名字}
获取方式：{PDF/仓库/URL/本地文件}
体量基准：{行数/页数/章节数}
标志性特征：{列 5-10 个标杆的标志性特征，用于后续逐项对照}
独特优势：{标杆有而一般产出没有的独家内容/能力}
```

**反面案例**：
- ❌ "达到行业最佳实践水平"（无法量化，无法判断达标）
- ❌ "像 OpenAI 的产品一样"（哪个产品？哪个维度？）
- ✅ "达到花叔橙皮书的质量"（75页 PDF 可获取，10章可数，特征可对照）

### ② 诊断差距（Diagnose）模板

**量化对比表**：
```
| 维度 | 当前产出 | 标杆 | 差距 | 影响 |
|------|---------|------|------|------|
| 体量（行数） | 400 | 1500 | -73% | 深度不足 |
| 章节数 | 8 | 10+附录 | -2 | 覆盖不全 |
| 特征A（如争议层） | 0 | 2 | -2 | 缺真实权衡 |
| 特征B（如心智模型） | 无 | 三层模型 | 缺 | 无导航框架 |
| ... | ... | ... | ... | ... |
```

**定性对比**（风格/温度/独家）：
```
风格语气：当前 {描述} vs 标杆 {描述} → 差距 {描述}
第一人称温度：当前 {描述} vs 标杆 {描述} → 差距 {描述}
独家内容：标杆有 {列出}，当前 {有/无} → 可复制/不可复制
```

**输出**：差距清单（每条标注量化值 + 影响 + 是否可改进）。

**关键原则**：差距必须量化。"深度不够"不算差距，"D6 争议层 0 章节 vs 标杆 2 章节"才算。

### ③ 对抗审查（Adversarial）模板

**派攻击者**（建议 3 个，各一视角）：

```
攻击者 A（可行性视角）：
  专攻：方案能落地吗？工具依赖/预算/技术约束会崩吗？
  找：硬依赖无降级、上下文爆炸、卡死场景

攻击者 B（质量视角）：
  专攻：改进真能达到标杆吗？还是会"形似神不似"？
  找：伪深度、兑水、同质化、幻觉放大

攻击者 C（一致性视角）：
  专攻：和现有机制冲突吗？自相矛盾吗？
  找：门禁膨胀、原则冲突、推翻已有教训
```

**每个攻击点独立验证**：
- 攻击者提出攻击点（标题/严重度/证据/影响/修复）
- 验证者独立判断：isValid（是否真成立）+ adjustedSeverity（修正严重度）+ mustFix（是否阻断）
- 只保留 isValid=true 的攻击

**修复优先级**：CRITICAL > MAJOR(mustFix) > MAJOR > MINOR（建议采纳）。

> 可用 Workflow 工具并行跑 3 个攻击者 + 验证阶段（参考 cc-2pp 的 Judge Panel 模式）。

### ④ 分轮迭代（Iterate）模板

**轮次规划**（从表到里）：
```
第 1 轮：风格层（低风险高收益）
  目标：让产出"看起来像标杆"
  典型改动：叙事化、对比框、建议框、语气调整
  验收：标志性风格特征命中数

第 2 轮：深度层（中风险）
  目标：让产出"内涵像标杆"
  典型改动：原理/演变/心智模型/争议/方向等深度维度
  验收：深度维度命中数 + 防兑水测试

第 3 轮：针对性改进（基于同题对照）
  目标：补标杆有而自己缺的具体点
  典型改动：教程实战、引言温度、量化数据
  验收：对照差距清单逐条消缺

第 4 轮：体量/篇幅
  目标：对齐篇幅（最易被忽视的瓶颈）
  典型改动：篇幅策略修正、每概念展开模板
  验收：体量达到标杆 X%
```

**每轮明确三件事**：
1. 这轮解决什么维度？（单一维度，不混）
2. 验收标准是什么？（量化）
3. 什么情况算完成？（达标线）

### ⑤ dogfooding 验证（Validate）模板

**用真实场景跑一遍**，产出量化验收报告：

```
=== 第 N 轮 dogfooding 验收 ===
测试场景：{用真实输入跑一遍}
体量：{行数} 行（标杆 {行数}，{百分比}%）
章节：{章节数}（标杆 {章节数}）

特征覆盖（逐项打分）：
[ ] 特征 A：命中 {N} 次
[ ] 特征 B：命中 {N} 次
...

深度维度：
[ ] D3 原理：{N} 处
[ ] D4 演变：{N} 处
...

防伪指标：
[ ] 引用 URL 数：{N}（验证可访问）
[ ] 修正清单命中：{N}（防回归）
[ ] 防幻觉机制触发：{N}

结论：达标 / 不达标（不达标→继续迭代，不进下一轮）
```

**关键**：没有量化验收报告，不算完成一轮。

### ⑥ 同题对照（Compare）模板

**同输入对照**（发现自评漏掉的真实差距）：
```
主题：{同一个主题}
标杆处理方式：
  - {标杆怎么开篇}
  - {标杆怎么讲概念}
  - {标杆怎么举例}
  - {标杆怎么收尾}
  - {标杆的独家内容}

自己处理方式：
  - {自己怎么开篇}
  - ...

差距分析：
  - 缺什么：{标杆有而自己无}
  - 深度够吗：{标杆做透而自己没透}
  - 结构性差距：{标杆的独家优势，自己能否复制}
    → 可复制：{怎么补}
    → 不可复制：{诚实承认边界}
```

**输出**：新一轮「针对性改进清单」，触发第④步下一轮。

### ⑦ 防回归固化（Harden）模板

**三种固化方式**：

1. **修正清单**（防重犯历史坑）：
```
### {领域/场景}（生成时必查）
| 坑 | 正确做法 | 来源 |
|----|---------|------|
| {坑描述} | {正确做法} | 第 N 轮对抗 C/M |
```

2. **门禁/护栏**（把标准做成检查项）：
```
- 心智模型过"反直觉测试"（换成目录读者预期会变吗？）
- 数字附"三件套"（值+来源URL+口径）
- 引言"双源核实"（WebSearch + WebFetch 逐字）
```

3. **版本演进记录**（写在技能末尾）：
```
## v{版本} 改造记录（{标杆} 验证）
**第 N 轮**：{改了什么}。✅ dogfooding：{场景} {行数}行，{量化结果}。
**关键发现**：{这轮最重要的洞察}
```

## 量化方法库

### 体量量化
- 行数（`wc -l`）
- 章节数（`grep "^## "`）
- 字数 / 估算阅读时间

### 特征覆盖量化
- 标志性特征逐项 `grep -c` 计数
- 覆盖率 = 命中特征数 / 总特征数

### 深度量化
- 深度维度命中次数（D3 原理/D4 演变/D5 心智模型/D6 争议/D7 方向）
- 每个概念的展开深度（句数 / 是否含痛点+原理+操作+边界+建议）

### 防伪量化
- 引用 URL 数（`grep -oE "https?://[^ )]+" | wc -l`）
- 修正清单命中数
- 防幻觉机制触发数（数字三件套/引言双源）

### 对比基准
- vs 标杆百分比（当前 / 标杆）
- 进步幅度（改后 / 改前）

## 避坑指南

### 坑 1：标杆模糊
**表现**：用户说"达到 XX 水平"，但 XX 不具体。
**防法**：第①步强制问清标杆三要素（具体/可量化/可获取）。不具体就不进第②步。

### 坑 2：跳过对抗直接改
**表现**：诊断完差距，直接动手改。
**防法**：第③步必走——方案再有道理，不经过对抗验证就是空中楼阁。9 个硬伤都是"听起来对，落地会崩"。

### 坑 3：一次改所有维度
**表现**：风格+深度+体量一起改。
**防法**：第④步分轮——一轮一维度。否则无法判断哪个改动有效，改坏了不知道回滚哪个。

### 坑 4：自评代替 dogfooding
**表现**：改完自己看一遍觉得"挺好"。
**防法**：第⑤步必跑真实场景 + 量化验收报告。没有报告不算完成。

### 坑 5：不做同题对照
**表现**：只自评，不拿标杆比自己。
**防法**：第⑥步必做——同题对照是发现真实差距的最强手段。自评永远偏乐观。

### 坑 6：教训不固化
**表现**：改完就结束，下次还犯同样的错。
**防法**：第⑦步必固化——修正清单/门禁/版本记录。不写进机制 = 下次还会犯。

### 坑 7：形式套件代替深度
**表现**：加了框/层/模板就觉得达标。
**防法**：每个深度层过"实质测试"——删了读者损失判断力吗？没损失就是形式主义。

### 坑 8：主动压制篇幅
**表现**：写"精简""删除测试"，执行成"能短则短"。
**防法**：篇幅指令明确"该详则详"，用标杆体量做锚点。

## 实战案例：cc-bestpractice → 橙皮书

> 这是 cc-benchmark 方法论的完整实证。记录了把 cc-bestpractice 技能从 v1.10.6（参考手册风格）迭代到 v1.11.2（橙皮书级深度）的全过程。

### 背景

- **被迭代对象**：cc-bestpractice 技能（领域最佳实践文档生成器）
- **标杆**：花叔《Claude Code 从入门到精通》橙皮书（75 页，10 章 + 附录）
- **初始状态**：v1.10.6，产出约 400 行精简文档
- **目标**：产出达到橙皮书的质量（风格 + 内容深度）

### 7 步执行实录

#### ① 定标杆
- 标杆对象：花叔橙皮书
- 获取方式：用户提供飞书链接 → 转 PDF → pdftotext 提取全文
- 体量基准：~1500-2000 行（75 页）
- 标志性特征：金句开路 / 痛点先行 / 推荐-不推荐对比 / 核心建议框 / 叙事反模式 / 三层心智模型 / 量化证据 / 人物引言 / 底层原理 / 演变脉络 / 争议权衡 / 方向感

#### ② 诊断差距
量化对比（初始 v1.10.6 产出 vs 橙皮书）：
| 维度 | v1.10.6 | 橙皮书 | 差距 |
|------|---------|--------|------|
| 体量 | 400 行 | ~1500-2000 行 | -73% |
| D6 争议层 | 0 | 2+ | 缺 |
| D7 方向层 | 简略 | 详尽 | 不足 |
| 第一人称温度 | 无 | 丰富 | 缺 |
| 完整项目实战 | 无 | §03 手把手 | 缺 |

#### ③ 对抗审查
派 3 攻击者（可行性/质量/一致性），13 个攻击点，验证后 9 个 mustFix：
- M2/M4 工具硬依赖无降级（CRITICAL）
- Research 不分阶段会爆炸（MAJOR）
- D1 量化诱发幻觉（MAJOR）
- D2 引言自废门禁 8（MAJOR）
- 深度层模板无落点（MAJOR）
- ...（共 9 个）

全部修复后才进第④步。

#### ④ 分轮迭代（4 轮）

**第 1 轮：风格层（v1.11.0）**
- 模式分流（不强统一）
- 橙皮书 5 步写作流
- 六件套（+推荐框/核心建议框）
- 叙事反模式
- dogfooding：FastAPI 461 行 ✅ 达橙皮书 75%

**第 2 轮：深度层（v1.11.0 续）**
- D3-D7 深度层（按文档类型选含）
- 信息源升级 7 类（配降级路径）
- Research 分阶段防爆炸
- 心智模型反直觉测试防兑水
- dogfooding：EDA 架构指南 269 行 ✅

**第 3 轮：针对性改进（v1.11.1，基于同题对照）**
- 教程模式（带端到端项目）
- D2 引言护栏放宽（作者授权经验）
- D1 量化强化（主动找基准）
- dogfooding：Claude Code 457 行 ✅ 达橙皮书 90%

**第 4 轮：体量对齐（v1.11.2）**
- 发现根因：之前"能短则短"反篇幅倾向是错的
- 改为"该详则详，把每个点讲透"
- 每概念 1-2 页展开模板
- dogfooding：Claude Code **1801 行** ✅ 达橙皮书 95%

#### ⑤ dogfooding 验证（每轮）
每轮产出量化验收报告（行数/章节/特征命中/深度维度/URL数/修正清单命中）。不达标不进下一轮。

#### ⑥ 同题对照（第 4 轮触发第 3 轮）
用 Claude Code 仓库生成文档，对比橙皮书发现 3 个真实差距：
- 陪练感不足（缺完整项目实战）→ 触发教程模式
- 第一人称温度不足 → 触发引言护栏放宽
- 量化数据偏少 → 触发量化强化

#### ⑦ 防回归固化
- OpenClaw 修正清单 5 个坑写进 SKILL.md（C1/C2/C4/M1/M3）
- 数字三件套 / 引言双源 / 反直觉测试 写成门禁
- v1.11.x 版本演进记录写进 SKILL.md 末尾
- 最终验证：OpenClaw 文档 1160 行，修正清单 9 处命中 ✅

### 进化曲线

```
v1.10.6（起点，400 行）—— 基准
   ↓ 对抗审查（9 硬伤修正）
v1.11.0（风格+深度，400-461 行）—— 75%
   ↓ 同题对照（发现 3 差距）
v1.11.1（教程+引言+量化，457 行）—— 90%
   ↓ 篇幅策略修正（反深度倾向）
v1.11.2（橙皮书体量，1801 行）—— 95%
```

### 最深刻的洞察

**第 4 轮的发现最关键**：之前 cc-bestpractice 最大的问题，不是"写不出深度内容"，而是**篇幅策略在主动压制深度**——"深度靠密度不靠字数""删除测试"听起来合理，执行时变成"能短则短"，导致产出永远只有橙皮书的 27%。

橙皮书的精神是**"该详则详，把每个点讲透"**。修正这个方向后，体量自然从 400 行跃升到 1801 行——不是堆字数，是每个概念都按完整结构展开。

### 残留差距（诚实承认边界）

- **独家源码分析**：橙皮书附录 A 的 51 万行源码分析，靠 web 搜不到，是花叔独家人脉研究。生成器难及。
- **真人第一人称厚度**：花叔"小猫补光灯做到 Top1"等独家人生经历，是"借素材"非"活出来"。

这 ~5% 是生成器的结构性限制，是合理的边界。

### 方法论验证结论

这次实战证明了 cc-benchmark 7 步闭环的有效性：
- **量化优先**：每轮都有数字证明进步（400→461→457→1801）
- **对抗找硬伤**：9 个"听起来对但落地崩"的硬伤被提前发现
- **分轮聚焦**：4 轮各解决一个维度，不混乱
- **同题对照**：发现了自评漏掉的 3 个真实差距
- **防回归**：OpenClaw 验证时修正清单 9 处命中，历史坑没重犯

## 与其他 cc-* 技能的关系

| 技能 | 解决什么 | 与 cc-benchmark 的关系 |
|------|---------|----------------------|
| cc-2pp | 新决策（多方案→对抗→裁决） | 对抗理念来源（Judge Panel） |
| cc-loop | 持续执行（循环自动化） | 迭代理念来源（闭环反馈） |
| cc-bestpractice | 产出文档（最佳实践生成） | 本次被迭代的对象 |
| **cc-benchmark** | **把已有产出提升到标杆水平** | **对标驱动迭代的元方法** |

cc-benchmark 是元技能——它不是产出某个具体东西，而是**指导"怎么把东西迭代到标杆"**。可以用来迭代任何产出（技能/文档/工具/流程）。
