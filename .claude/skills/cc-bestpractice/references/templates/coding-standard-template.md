## 3. 命名规范

<!-- 填充指令：覆盖至少 5 类元素：变量/函数/类/常量/模块/接口/枚举/泛型参数。
     每类一条规则 + 正例 + 反例。命名相关断言标来源（PEP 8/官方风格指南）。 -->

### 3.1 变量与参数命名

<!-- 占位格式：
     规则：变量用 <snake_case/camelCase> [置信度:★★★] [来源:<PEP 8>]
       正例：`user_count`
       反例：`userCount` / `uc`（来源：<lint 规则>）
-->

### 3.2 函数与方法命名

<!-- 占位格式：
     规则：函数名用 <snake_case>，动词开头表达意图 [置信度:★★★]
       正例：`def fetch_user(user_id: int) -> User:`
       反例：`def data(id):`（名词无意图，来源：<社区风格>）
     规则：私有方法前缀 <_> [置信度:★★★]
       正例：`def _validate(self):`
       反例：`def validate_private(self):`（来源：<官方命名约定>）
-->

### 3.3 类与接口命名

<!-- 占位格式：
     规则：类名用 <PascalCase> [置信度:★★★] [来源:<PEP 8>]
       正例：`class UserService:`
       反例：`class userService:` / `class user_service:`
     规则：接口/抽象类前缀或后缀约定（如 I 前缀 or ABC 后缀，二选一） [置信度:★★]
       正例：`class UserRepository(ABC):`
       反例：混用两种约定
-->

### 3.4 常量命名

<!-- 占位格式：
     规则：模块级常量用 <UPPER_SNAKE_CASE> [置信度:★★★]
       正例：`MAX_RETRY = 3`
       反例：`maxRetry = 3`（来源：<PEP 8>）
-->

### 3.5 模块与文件命名

<!-- 占位格式：
     规则：模块文件名用 <lowercase_with_underscores>，禁中划线（来源：<PEP 8>） [置信度:★★★]
       正例：`user_service.py`
       反例：`userService.py` / `user-service.py`（中划线 → import 报错）
-->

### 3.6 其他元素（枚举/泛型/类型别名）

<!-- 占位格式：
     规则：枚举成员用 <UPPER_SNAKE> 或 <PascalCase>（二选一，统一） [置信度:★★]
       正例：`Color.RED`
       反例：`Color.red`
     规则：泛型类型参数用 <T> / <T_co> 等 <单字母+下划线后缀>（来源:<PEP 484>） [置信度:★★★]
       正例：`def get[T](items: list[T]) -> T:`
       反例：`def get ItemType)(items):`
-->

## 4. 语句规范

<!-- 填充指令：覆盖至少 5 类语句：赋值/控制流/循环/异常/导入。
     每类一条规则 + 正例 + 反例。 -->

### 4.1 赋值语句

<!-- 占位格式：
     规则：禁链式赋值给不同值；禁赋值做条件 [置信度:★★★]
       正例：`x = 0; y = 0`（明确）
       反例：`x = y = 0` 当意图不同时；`if x = func():`（来源：<lint 规则>）
     规则：多目标解包用元组解包 [置信度:★★★]
       正例：`x, y = 1, 2`
       反例：`temp = 1; x = temp; y = temp`
-->

### 4.2 条件语句

<!-- 占位格式：
     规则：条件用显式布尔比较，禁隐式真值判断（除明确判断 None/空集合） [置信度:★★]
       正例：`if flag is True:` 或 `if flag:`
       反例：`if flag == True:`（来源：<lint E712>）
     规则：禁嵌套 > <N> 层 if（用卫语句/提前返回） [置信度:★★★]
       正例：卫语句 `if not valid: return`
       反例：`if a: if b: if c: ...` 深嵌套（来源：<社区重构指南>）
-->

### 4.3 循环语句

<!-- 占位格式：
     规则：禁在循环中修改正在迭代的集合 [置信度:★★★] [来源:<官方语言规范>]
       正例：先收集变更 → 循环后统一应用
       反例：`for x in items: items.remove(x)`
     规则：优先用 comprehension 替代 map/filter + lambda（可读性） [置信度:★★★]
       正例：`[f(x) for x in items if x > 0]`
       反例：`list(map(lambda x: f(x), filter(lambda x: x > 0, items)))`
-->

### 4.4 异常语句

<!-- 填充指令：异常相关规则是反例门禁的重点检查区，必须列"裸 except / 吞异常 / 异常泛化"三条反例。 -->
<!-- 占位格式：
     规则：捕获具体异常类型，禁裸 except / 禁 except Exception 吞所有 [置信度:★★★] [来源:<官方 tutorial>]
       正例：`except ValueError as e: log(e); raise`
       反例：`except: pass`（吞掉 KeyboardInterrupt/SystemExit）
     规则：异常重新抛出时禁裸 raise 之外的吞栈（用 `raise` 不用 `raise e`） [置信度:★★★]
       正例：`except X: handle(); raise`
       反例：`except X as e: raise e`（丢失原始 traceback）
     规则：禁用异常做控制流（性能差且语义错） [置信度:★★]
       正例：`if key in d: ...`
       反例：`try: d[key] except KeyError: ...` 当常规路径（来源：<社区性能指南>）
-->

### 4.5 导入语句

<!-- 占位格式：
     规则：禁 `from x import *`（污染命名空间） [置信度:★★★] [来源:<PEP 8>]
       正例：`from x import specific_name`
       反例：`from x import *`
     规则：导入顺序三段式（标准库 → 第三方 → 本地），组间空行 [置信度:★★★]
       正例：
       ```python
       import os

       import requests

       from myapp import utils
       ```
       反例：三类导入混杂无空行（来源：<isort/官方>）
-->

## 5. 注释与文档

### 5.1 注释规范（何时注释 + 格式）

<!-- 填充指令：区分"为什么"注释（保留）vs"是什么"注释（删，代码自解释）。
     给注释格式约定（# 风格 / 文档型 / TODO 格式）。 -->
<!-- 占位格式：
     规则：注释解释"为什么"，不解释"是什么"（代码已说明是什么） [置信度:★★★]

     规则：TODO 格式：`# TODO(<owner>): <描述>` [置信度:★★]
       正例：`# TODO(alice): 替换为批量接口，2026Q3`
       反例：`// fix later`（无负责人无上下文）
-->

### 5.2 文档字符串 / Doc 规范（格式 + 示例）

<!-- 填充指令：选一种 docstring 风格（Google/Sphinx/NumPy，二选一统一）+ 公开 API 必写 docstring。
     给完整可复制的 docstring 模板占位（含参数/返回/异常/示例）。 -->
<!-- 占位格式：
     规则：公开函数/类/模块必须有 docstring，风格统一 <Google/NumPy/Sphinx> [置信度:★★★] [来源:<PEP 257>]
       正例模板（Generate agent 选定风格后填）：
       ```python
       def fetch_user(user_id: int) -> User:
           """根据 ID 获取用户。

           Args:
               user_id: 用户唯一标识，正整数。

           Returns:
               User 对象。

           Raises:
               NotFoundError: 用户不存在时。

           Example:
               >>> fetch_user(1)
               User(id=1, name='alice')
           """
       ```
       反例：`def f(x):` 无 docstring 的公开 API（来源：<lint D103>）
     规则：模块级 docstring 写在文件首行，说明模块职责 [置信度:★★★] [来源:<PEP 257>]
       正例：`"""用户服务：用户 CRUD 与权限校验。"""`
       反例：模块首行直接是 import
-->

## 6. 禁止清单

<!-- 填充指令：至少 10 条，每条四列齐全 + 严重度（高/中/低）。
     禁止"不要写烂代码"式废话（违反门禁4）。每条尽量带 lint 规则编号或 PEP/CWE 编号。
     严重度取值：高（安全/正确性/数据丢失）/ 中（可读性/可维护性）/ 低（风格统一）。
     下表已给 12 条占位，Generate agent 替换为领域实际禁止项，可增删但必须 ≥ 10 条。 -->

| # | 禁止项 | 原因 | 替代方案 | 严重度 |
|---|--------|------|---------|--------|
| 1 | 裸 `except:` / `except Exception: pass` | 吞掉所有异常含 KeyboardInterrupt/SystemExit，隐藏 bug | `except SpecificError as e: log(e); raise` [来源:<官方 tutorial>] | 高 |
| 2 | `from x import *` | 污染命名空间，遮蔽本地名，无法静态分析 | 显式 `from x import name1, name2` [来源:<PEP 8>] | 高 |
| 3 | f-string / 字符串拼接 SQL | SQL 注入 | 参数化查询 `cursor.execute(sql, (param,))` [来源:<CWE-89>] | 高 |
| 4 | 硬编码密钥/URL/路径进代码 | 泄密风险 + 多环境难切换 | 环境变量 + 启动期校验 [来源:<12factor.net>] | 高 |
| 5 | 可变默认参数（`def f(x=[])`) | 默认值在函数定义时求值一次，跨调用共享状态 | `def f(x=None): x = [] if x is None else x` [来源:<官方 FAQ>] | 高 |
| 6 | 全局可变状态（模块级 dict/list 当缓存） | 并发不安全 + 难测试 | 显式生命周期 + 依赖注入或缓存库 [来源:<社区 issue>] | 中 |
| 7 | 赋值做条件（`if x = func():`) | 混淆赋值与比较，可读性差 | 分两行：`x = func(); if x:` [来源:<lint 规则>] | 中 |
| 8 | 魔法数字（无命名的裸数字） | 语义不明，改一处怕漏 | 提取为命名常量 `MAX_RETRY = 3` | 中 |
| 9 | 嵌套 > 3 层的控制流 | 可读性骤降，难测试 | 卫语句 / 提前返回 / 抽小函数 [来源:<社区重构指南>] | 中 |
| 10 | 注释掉的代码块提交 | 噪音，版本控制已留历史 | 直接删除，需找回用 `git log` [来源:<社区风格>] | 低 |
| 11 | 单字母变量名（除循环计数 i/j/k） | 语义不明 | 描述性名 `user_count` | 低 |
| 12 | tab 与空格混用缩进 | 跨编辑器显示错乱 | 统一 <N 空格>，formatter 强制 [来源:<PEP 8>] | 低 |
