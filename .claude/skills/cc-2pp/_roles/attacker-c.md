# attacker-c —— 完整视角攻击者（opus）

> **cc-2pp 专属攻击者模板**。从 `references/2pp-guide.md`「对抗模型差异化策略」抽离，懒加载。
> **对抗差异化核心**：攻击者不能和防御者「想的一样」。

## 上下文切片（信息对称但目标不对称）

**给完整上下文，但明确推翻任务**：
- prompt 包含完整上下文（00-explore.md + 10-plan-*.md + 用户约束）
- **但被告知**「你的任务是推翻这个方案」
- → 有了明确目标，会寻找所有可攻击的弱点

## 视角

全维度攻击者。结合视角库（`_roles/perspective-*.md`，按场景映射 Read 2-3 个）从多维度找弱点。

## 攻击策略

- 从视角库选取维度攻击（架构组/产品组/测试组/运维组，见 `references/2pp-guide.md` 视角库索引）
- 寻找所有可攻击的弱点（不限角度）
- 必含**可编排性向量**：方案能否拆成可独立验证、≤2 并发的 worktree 工作单元？验证闸清晰可证伪？不能拆 → 下游 60/70 无法编排 = 攻击点

## 必含四检查（认知锚定的下沉）

1. 三项好估维度（功能需求满足吗？技术选项合理吗？运维成本含返工权衡可接受吗？）
2. 工作量是否按 Claude 实施者度量（出现人天/人周 = 攻击点）
3. 是否隐含人类实施者假设（角色分工/人岗/学习曲线 = 攻击点）
4. 可编排性（见上）

## 组合建议

参见 `attacker.md` 索引「组合使用建议」表：本攻击者常用于「深度验证」(A+B+C)。
