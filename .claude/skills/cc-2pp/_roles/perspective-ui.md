# perspective-ui —— UI 视角（sonnet）

> **cc-2pp 专属视角模板**。从 `references/2pp-guide.md`「视角库」抽离，懒加载。

## 关注维度

- 用户体验: 操作路径是否最短？
- 可访问性: 残障用户能否使用？
- 设计系统一致性: 是否复用已有组件？
- 响应式: 不同设备表现如何？
- 加载性能: 首屏/交互响应是否达标？

## 攻击策略

- "用户完成核心任务需要几步？能不能更少？"
- "没有鼠标/键盘/屏幕的用户能使用吗？"
- "这个设计在手机上会变成什么样？"

## 评分标准（权重）

- 用户体验: 30%
- 设计一致性: 25%
- 可访问性: 15%
- 响应式: 15%
- 加载性能: 15%

## UIUX 设计检查清单（7 条，2026-06-29）

UIUX 决策时过这 7 条（按需——UIUX 决策触发，纯后端跳过）：

- **A 用户旅程完整**：核心任务全路径，含错误/空/加载态，不只 happy path（反只画 happy path）
- **B 一致性**：设计系统统一，复用组件，反重复造（反碎片化）
- **C 反馈及时**：操作有响应（loading/success/error/disabled），反无响应
- **D 容错与恢复**：错误可恢复 / 撤销 / 危险操作确认，反不可逆
- **E 可访问性**：a11y（键盘 / 屏幕阅读器 / 对比度 / 焦点），非可选
- **F 性能感知**：加载 / 响应 / 骨架屏，用户感知优于实际
- **G 可学习性**：新人上手成本，认知负载，反隐晦

**建议执行序**（cc-2pp 对抗验证裁决 b：原则的排序，不重复内容，按需触发——UIUX 决策才走，纯后端跳过）：
① A 用户旅程完整 → ② B 一致性 → ③ C 反馈 → ④ D 容错 → ⑤ E 可访问性 → ⑥ G 可学习性 → ⑦ F 性能感知
