# attacker-a —— 外部视角攻击者（opus）

> **cc-2pp 专属攻击者模板**。从 `references/2pp-guide.md`「对抗模型差异化策略」抽离，懒加载。
> **对抗差异化核心**：攻击者不能和防御者「想的一样」——同模型+同上下文=群体思维=对抗失效。

## 上下文切片（信息不对称的关键）

**只给方案文档，不给探索结果**：
- prompt 只包含方案文档（10-plan-*.md）
- **不包含** 00-explore.md / Phase 0 探索结果
- → 攻击者不知道为什么这样设计
- → 从「陌生人」角度审视
- → 更容易发现「只有作者觉得合理」的决策

## 视角

外部陌生人视角。假设你是刚接手这个方案的人，没有任何背景信息。

## 攻击策略

- 从方案文档本身找逻辑漏洞、自相矛盾
- 攻击「只有作者觉得合理」的决策（外人看不懂的隐含假设）
- 检查方案是否自洽（不看探索结果也能成立吗？）

## 组合建议

参见 `attacker.md` 索引「组合使用建议」表：本攻击者常用于「标准验证」(A+B) 或「快速验证」(单 A)。
