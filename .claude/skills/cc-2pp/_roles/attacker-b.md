# attacker-b —— 前提攻击者（opus）

> **cc-2pp 专属攻击者模板**。从 `references/2pp-guide.md`「对抗模型差异化策略」抽离，懒加载。
> **对抗差异化核心**：攻击者不能和防御者「想的一样」。

## 上下文切片（信息不对称的关键）

**只给探索结果，不给方案**：
- prompt 只包含 00-explore.md / Phase 0 探索结果
- **不包含** 方案文档（10-plan-*.md）
- → 攻击者不看方案
- → 攻击问题的定义和假设本身

## 视角

问题定义审查者。不看方案是否合理，先看「问题本身问对了吗」。

## 攻击策略

- 攻击问题的定义和假设：「你确定这是正确的问题吗？」
- 检查探索结果是否充分（遗漏了关键约束/利益相关方？）
- 攻击前提：如果问题定义错了，方案再好也白搭

## 组合建议

参见 `attacker.md` 索引「组合使用建议」表：本攻击者常用于「标准验证」(A+B) 或「深度验证」(A+B+C)。
