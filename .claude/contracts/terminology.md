# terminology — hcc 术语精确化表（去模糊化单一来源）

> 配套 `charter.md` §1 总则6。所有部门产出（skills/contracts/scripts/decisions 文档）的术语可证伪定义。
> cc-2pp injection-template 注入「禁令清单」+ hcc-preflight.js 扫描「禁令清单」均引用本文件。
> 来源：2026-06-28 terminology-deblur 决策（γ 三件套·执行层）。

## 判据

术语**可证伪** = 能写成「命令 + 期望输出」或「可枚举检查项」。不能 → 模糊词，必须替换或补操作定义。

## 使用方式

- 写产出前：查本表，禁用「纯模糊」词，用「替换形态」替代
- agent 起草/攻击/计划：injection-template 注入「§禁令清单」
- 校验：hcc-preflight.js grep「§禁令清单」（反例/禁令语境除外）

## A. 态度/程度词（纯模糊 → 替换为行为清单/阈值）

| 现词 | 替换形态（可证伪） | 可枚举项 |
|---|---|---|
| 最懒（P1） | **用户操作成本最低**：判据「让用户更省事 vs 让设计者更少做」，主语必须是用户 | 主语检查 |
| 努力 / effort=max | **步骤完整性清单** | 4 条：Explore0a 必做 / Plan 用 opus / 攻击者≥3 全 opus / Phase4 必做 |
| 充分（探索） | **探索完备性** | 0a 内部 + 0b 外部 ≤15 轮 + 覆盖{关键文件/架构约束/已有决策/能力清单}4 维度 |
| 拼命（挖需求） | **需求挖掘度** | 问 ≥2-3 侧面 + web 补齐 + 三项好估澄清 |
| 深入（分析） | 删除，改具体维度（如「按性能/生态/运维 3 维度分析」） | 维度可枚举 |
| 尽快（完成） | 删除（无时间预算语义），或绑具体 SLA | — |
| 合理（疑问句） | 改可证伪提问「X 是否满足 Y 标准？」；禁裸「是否合理」 | grep 裸句式 |
| 适当 | 已在 cc-2pp/cc-goal 禁令清单，本表扩展到全体系 | grep |

## B. 比喻词

### B1 纯模糊（删或补操作定义）

| 现词 | 处理 | 操作定义 |
|---|---|---|
| 空中楼阁 | 删除（无信息量） | — |
| 灵魂（循环的灵魂） | 删除（纯修辞） | — |
| 一等公民 | 补定义 | agent 写文件 + 编排者读文件全文做判断；两类 agent 权限表（探索类只读/创造类写） |
| 保姆级（需求清单） | 补定义 | 每条 R{n} 含 6 字段（做什么/输入/输出/**可证伪验证**/依赖/预估），验证=命令+期望 |
| 真综合（vs 拼接） | 补定义 | 读 N 个 agent 产出**文件全文**判断，禁返回值摘要；30-score/40-synthesis 引用各 10-*/20-* 文件路径 |

### B2 半模糊（补可证伪项）

| 现词 | 现状 | 补充判据 |
|---|---|---|
| 闭环（vs 开环） | 有动作链（写→测→读→修→重复） | 补三件：触发器 + 失败信号 + 终止条件 |
| 漂移（认知/引用） | 「偏离目标」无度量 | 补检测阈值（如约束在第 K 轮消失 = 漂移信号） |
| 锚点（关键信息） | 部分操作化（锚文件已定义） | 补锚点判定标准（哪些信息必锚：终态/约束/决策） |
| 系统性（缺陷/失败） | 部分（8/8 失败=系统性） | 补比例阈值（≥N 个同源失败 = 系统性） |

### B3 已操作化（名留，定义确认）

| 词 | 操作定义 |
|---|---|
| 护栏三件套 | max_iteration + no_progress_streak + budget_tokens_cap（3 条可枚举） |
| 锚文件（固定重读） | VISION/CLAUDE/AGENTS/PROMPT，每次迭代重读 |
| 语义锚名 | 锚名即 API，改锚名 = bump version + grep 同步 |
| 验证闸 | 命令 + 期望输出（可证伪对） |
| 可证伪闸 | 脚本断言（location∈枚举 + required_for 数组等） |
| 加严（验证闸） | plan mode：每步必跑 + 多跑 1 轮 smoke + 断言数 ≥ 2pp 的 1.5x |
| 更细（步骤） | 每步 ≤ 1 个可独立验证单元 |

## C. 相对词（纯模糊 → 绑标尺）

| 现词 | 标尺 |
|---|---|
| 轻量 / 深度（mode 总开关） | **M2 特征向量 score**：≥5→2pp / ≤2→plan（非「轻/重/深」直觉） |
| 完整度（评分） | Phase0 约束逐条回应数 / 总约束数 = 比例 |
| 足够（测试覆盖） | 绑具体阈值（如分支覆盖率 ≥ X%） |
| 快速（plan/验证/回滚） | 绑时间预算或删除 |
| 最快（交付） | 缺陷反例语境，改「在 X 维度可比较的快」 |
| 简洁（部署） | 评分维度，绑可枚举检查项 |

## D. 极致词（→ 指明可比较维度）

| 现词 | 替换 |
|---|---|
| 最优 / 次优 | 「在三项好估维度上得分最高/次高」（评分驱动） |
| 最佳（实践/工具） | 删除或「在 X 场景下经验证优」 |
| 完美 | 删除（反例语境） |
| 最快 | 见 C |

## §禁令清单（preflight 扫描 + injection 注入用）
